# Security Policy

## Supported Versions

We actively maintain the latest major version of Resultify.

Older major versions may not receive security updates.

| Version | Supported |
|---------|-----------|
| Latest  | ✅ Yes     |
| Older   | ❌ No      |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it responsibly by:

- Opening a GitHub Security Advisory (preferred)
- Or contacting the maintainer directly (if an email is provided)

Please include:

- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested mitigation (if known)

---

## Disclosure Policy

- Security reports will be reviewed as quickly as possible.
- If confirmed, a fix will be prepared.
- A new release will be published addressing the issue.
- Public disclosure will occur after a fix is available.

We appreciate responsible disclosure and will credit reporters when appropriate.
